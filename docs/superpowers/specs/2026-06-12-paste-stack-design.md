# Paste Stack(顺序粘贴队列)设计文档

日期:2026-06-12
状态:已与用户确认

## 目标

把多条剪贴板历史按顺序逐条粘贴到目标应用(典型场景:把姓名/电话/地址逐个填进表单的不同输入框),全程只使用用户已熟悉的 Cmd+V,不引入新粘贴快捷键。

## 用户交互流程

### 入队

- 在历史浮窗(Cmd+Shift+V)中,对选中条目按 **Space**(或点击行尾 ⊕ 图标)加入队列。
- 入队条目行尾显示序号徽标(①②③…),顺序 = 入队顺序(FIFO),v1 不支持队列内重排。
- 重复按 Space 可将已入队条目移出队列(后续序号自动收缩)。
- 队列非空时,浮窗以任意方式关闭(ESC、失焦)即启动队列,HUD 出现;误入队直接在 HUD 点 ✕ 取消即可。队列为空则关闭浮窗无任何副作用。

### 粘贴(核心机制:预写 + 被动观察)

1. 队列启动时,把 ① 写入系统剪贴板(经 PasteboardSelfWriteGate 标记自写)。
2. 用户在任意目标应用按**普通 Cmd+V**,系统正常完成粘贴——VeloxClip 不拦截、不注入,仅通过全局按键监视(NSEvent global monitor,复用已有的辅助功能权限)**旁观**到这次 Cmd+V。
3. 观察到 Cmd+V keyUp 后延迟约 100ms(给目标应用读剪贴板留时间),将下一条写入剪贴板,HUD 指针前进。写入前用 changeCount 校验剪贴板仍是我们写的内容,防止竞态跳条。
4. 重复 2–3 直到队列耗尽。

### 暂停规则

- 队列激活期间检测到**非队列写入**(用户复制了新内容):队列立即暂停,HUD 显示暂停态(▶ 继续 / ✕ 退出)。此时普通 Cmd+V 粘贴用户自己复制的内容,队列不抢戏。
- 点 ▶ 继续:把当前队列项重新写入剪贴板,恢复观察。
- 规则一句话:剪贴板被用户动过,队列就让路;用户说继续,它再接管。

### 结束(方案 A:一次性)

- 贴完最后一条:HUD 显示「✓ 完成」约 1 秒后淡出,模式结束。
- 中途退出:HUD 的 ✕ 按钮。
- 结束/退出时恢复剪贴板为**最后一次非队列写入**的内容:
  - 期间用户没复制过任何东西 → 恢复进入队列前的内容;
  - 期间复制过 X → 剪贴板保持 X(无需动作)。
- 区分"队列写的 / 用户写的"依赖现有 PasteboardSelfWriteGate 的 changeCount 标记,不引入新机制。

## HUD

- 形态:小型悬浮条(NSPanel,`.nonactivatingPanel`,`.floating` 层级,`canJoinAllSpaces` + `fullScreenAuxiliary`),与 PasteImageService 的悬浮窗同一套窗口机制。
- **绝不抢焦点**——出现与更新时用户光标始终留在目标输入框内,这是体验成立的前提。
- 默认位置:屏幕右下角;可拖动,拖动后位置持久化(设置中显示为"自定义")。
- 进行中(极简一条):`下一条:138-0000-0000  2/3`,每次前进有滚动反馈。
- 暂停态:显示「⏸ 已暂停(检测到你复制了新内容)」+ ▶ 继续 / ✕ 退出按钮。
- 完成态:「✓ 完成 3/3」1 秒后淡出。

## 设置(AppSettings 新增)

| 设置项 | 类型 | 默认 |
|--------|------|------|
| 显示队列 HUD | 开关 | 开 |
| HUD 位置 | 右下 / 左下 / 右上 / 左上(拖动后显示"自定义") | 右下 |

- HUD 关闭时:队列照常工作,改由**菜单栏图标旁显示进度文本**(如 `2/3`),队列结束后恢复正常图标。

## 架构

新增 `Services/PasteStackService.swift`(@MainActor 单例):

- 队列状态机:`idle → active ⇄ paused → completed → idle`。
- 持有队列条目快照(`[ClipboardItem]`,入队时若 `item.data == nil` 且为图片/RTF,经 `ClipboardStore.loadData(for:)` 按需加载后快照)。
- 剪贴板写入:复用 `ClipboardItem` 现有的 pasteboard 写入逻辑 + SelfWriteGate。
- 全局按键监视(Cmd+V keyUp)与剪贴板 changeCount 校验;监视器只在 active 状态存在,暂停/结束即移除。

新增 `Views/PasteStackHUD.swift`:HUD 的 NSPanel 包装 + SwiftUI 内容视图,读 PasteStackService 状态。

改动既有文件:

- `ClipboardListView` / `MainView`:Space 入队手势、⊕ 图标、序号徽标。
- `AppSettings` + `SettingsView`:两个新设置项。
- `VeloxClipApp`(菜单栏):HUD 关闭时的进度文本。

## 边界与错误处理

- 队列项支持 text / image / file / rtf / color(与现有"复制到剪贴板"能力一致);图片/RTF 入队时按需加载 blob。
- 右键菜单"粘贴"等非键盘粘贴无法被观察到,指针不前进——HUD 的 ▶ 可手动校准,属罕见路径,文档化即可。
- 粘贴极慢的目标应用存在理论竞态:changeCount 校验兜底,校验失败时不前进并保持当前项。
- 锁屏/切换用户:监视器收不到事件,队列保持原状,回来继续可用。
- 队列激活时再次打开历史浮窗:允许,正常浏览/粘贴单条会触发"非队列写入"→ 暂停,符合统一规则。

## 不做(YAGNI)

- 队列内拖拽重排、循环模式、"再来一轮"、边复制边入队(Copy Stack)、劫持 Cmd+V(CGEventTap)。

## 测试

- 单测(不依赖真实 pasteboard,状态机注入假写入器):入队/移出、FIFO 前进、暂停/恢复、完成、剪贴板恢复规则(中途有/无用户复制两种情形)、changeCount 校验失败不前进。
- 手测清单:跨应用连贴、中途复制暂停、HUD 拖动持久化、HUD 关闭时菜单栏进度、全屏应用上的 HUD 可见性。
